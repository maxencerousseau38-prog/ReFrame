> "When you’re running an SEO experiment you’re testing on the page level. Thus, you need to have an experimentation system that can do your bucketing on the page level."

**Insight:** SEO experiments must bucket by the page level rather than the user level to ensure search engine crawlers receive a consistent experience across every visit.

**Tactical advice:**
- Bucket treatment and control groups by taking a hash of the canonical URL.
- Build a custom visitors table to log every incoming visit and referral for accurate, unsampled data.
- Roll out changes site-wide across both mobile and desktop to ensure Googlebot receives a consistent experience regardless of which crawler it uses.

*Source: [How to run SEO experiments](https://www.lennysnewsletter.com/p/how-to-run-seo-experiments)*

> "You’ll want to let the experiment run for a while so that Google can discover the changes, and make the necessary adjustments. SEO experiments take more time than A/B tests to show impact."

**Insight:** Executing an SEO experiment requires a longer duration than standard A/B tests to account for search engine discovery and a 'difference in difference' analysis to manage baseline traffic imbalances.

**Tactical advice:**
- Run experiments for 2-4 weeks to allow sufficient time for Google to crawl and index changes.
- Use 'difference in difference' analysis to account for traffic imbalances between random page buckets.
- Exclude bot traffic from your success metrics by filtering out user agents from known crawlers like Googlebot and Bingbot.

**Insight:** SEO is a defensible growth engine only when the product supports programmatic scaling or a unique data angle that captures long-tail search intent.

**Tactical advice:**
- Analyze your category for 'make' or 'template' keyword opportunities that can be automated.
- Evaluate if you have unique internal data that can be used to generate thousands of search-indexed pages.
- Build a programmatic engine to capture design or project-specific traffic rather than relying solely on manual editorial content.
